# Claude Code + Codex: 異モデル批判レビューで質を上げる

Claude Code（Anthropic）の中から OpenAI Codex（GPT-5.4）を呼び出し、**書いたコードを別のAIに批判させる**ワークフローのガイドです。

## なぜこれをやるのか

| 課題 | 解決策 |
|---|---|
| 同じモデルが書いてレビューすると、同じ盲点を見逃す | **別モデル**にレビューさせて盲点を補完 |
| レビューや調査でClaudeのコンテキストが埋まる | Codexに委譲してClaudeの**トークンを節約** |
| レビューを待つ間、手が止まる | Codexをバックグラウンド実行しながらClaude側で作業続行 |

## 仕組み

```
┌───────────────────────────────┐
│  Claude Code (Opus)           │  あなたとの対話・設計・実装
│                               │
│  /codex:review ───┐           │
│  /codex:rescue ───┤           │
└───────────────────┼───────────┘
                    │  Bash 1回で転送（薄いブリッジ）
                    ▼
┌───────────────────────────────┐
│  codex-companion.mjs          │  Claude → Codex CLI への橋渡し
└───────────────────┬───────────┘
                    │
                    ▼
┌───────────────────────────────┐
│  Codex CLI (GPT-5.4)          │  独立コンテキストでレビュー・診断・修正
└───────────────────────────────┘
```

**ポイント**: Claudeのコンテキストには「Codexの出力結果」だけが返る。レビューの思考過程や中間ファイル読み込みはCodex側で完結するため、Claudeのトークン消費を大幅に抑えられる。

## セットアップ

### 前提条件

- [Claude Code](https://docs.anthropic.com/en/docs/claude-code) がインストール済み
- Node.js (npm) が使える環境

### 手順

```bash
# 1. Codex CLIをインストール
npm install -g @openai/codex

# 2. OpenAI認証
codex login

# 3. Claude Code内でセットアップ確認
/codex:setup
```

`/codex:setup` を実行すると、Codex CLIの状態・認証状況を自動チェックしてくれます。未インストールの場合はその場でインストールも可能です。

## コマンド一覧

### `/codex:review` — コードレビュー

gitの変更差分をCodexにレビューさせます。

```
/codex:review                        # ワーキングツリーの変更をレビュー
/codex:review --background           # バックグラウンドで実行
/codex:review --base main            # mainブランチからの差分をレビュー
/codex:review --scope branch         # ブランチ全体をレビュー
```

### `/codex:adversarial-review` — 敵対的レビュー

通常のレビューより踏み込んで、**設計判断そのものを批判**します。

```
/codex:adversarial-review                          # 全体を批判的にレビュー
/codex:adversarial-review 認証周りが不安           # フォーカス指定も可能
/codex:adversarial-review --background --base main  # 組み合わせも可
```

### `/codex:rescue` — 調査・修正の委譲

Claude側で手が止まった時や、重い作業を丸投げしたい時に使います。

```
/codex:rescue このテストが落ちる原因を調べて修正して
/codex:rescue --background 全エンドポイントのエラーハンドリングを見直して
/codex:rescue --resume                              # 前回の続きから再開
```

### `/codex:setup` — 状態確認

```
/codex:setup
```

## 実践ワークフロー

### 基本: 実装 → 批判レビュー → 修正

```
あなた:  「ユーザー認証のミドルウェアを実装して」
Claude:  （実装する）

あなた:  /codex:adversarial-review --background
         （Codexがバックグラウンドでレビュー中、Claudeで別作業を進める）

あなた:  /codex:status
         （レビュー完了を確認）

Codex:   「トークン検証でタイミング攻撃の可能性、
          リフレッシュトークンのローテーションが未実装」

あなた:  「Codexの指摘を踏まえて修正して」
Claude:  （指摘箇所を修正）
```

### 応用: Claudeが詰まったらCodexに投げる

```
Claude:  「このエラーの原因が特定できません…」

あなた:  /codex:rescue このエラーの根本原因を調べて修正して

Codex:   （独立コンテキストでファイルを読み、原因を特定、修正を実行）
         「原因: DBマイグレーションのカラム名不一致。修正済み。」
```

## adversarial-review の中身

敵対的レビューがなぜ効くのか。プロンプトの設計思想を解説します。

### レビュアーのスタンス

> **「この変更を出荷すべきでない理由を探せ」**

通常のコードレビューが「問題がないか確認する」のに対し、adversarial-reviewは「問題があると仮定して証拠を探す」アプローチです。

### 攻撃対象（何を見るか）

出荷後に高コストになる障害を優先的に探します:

- 認証・権限・テナント分離・信頼境界
- データ損失・破損・重複・不可逆な状態変更
- ロールバック安全性・リトライ・部分障害・冪等性
- レースコンディション・順序依存・古い状態・再入
- 空状態・null・タイムアウト・依存サービスの劣化
- バージョン差異・スキーマ変更・マイグレーション
- 障害を隠す、または復旧を困難にするオブザーバビリティの欠落

### 報告基準（何を報告しないか）

- スタイルや命名の指摘 → 対象外
- 根拠のない推測 → 対象外
- 「念のため」レベルの懸念 → 対象外

報告されるのは**具体的なコード行・現実的な障害シナリオ・影響度・修正案**がセットになったものだけです。

## トークン節約の効果

```
■ Claude単体で全部やる場合

  [ファイル読み込み]  ████████░░  トークン消費
  [実装]              ████████░░
  [レビュー用再読み]  ████████░░  ← ここが無駄
  [レビュー思考]      ██████████
  [修正]              ████░░░░░░
  ─────────────────────────────
  合計                ██████████████████████████████████████

■ Codexに委譲する場合

  [ファイル読み込み]  ████████░░  トークン消費
  [実装]              ████████░░
  [Codex結果受取]     ██░░░░░░░░  ← 結果だけ
  [修正]              ████░░░░░░
  ─────────────────────────────
  合計                ██████████████████████████
                      ↑ レビュー部分のコスト削減
```

特にファイル数が多いレビューほど差が開きます。Codex側のトークンは別課金（OpenAI側）なので、Claudeのコンテキスト枠を温存できます。

## なぜ「異モデル」が効くのか

**同一モデルのレビューには構造的な限界がある**。自分が書いたコードを自分でレビューする人間と同じで、同じ思考パターンで書いたものは同じ思考パターンでは見逃しやすい。

異なる学習データ・推論パターンを持つモデルに「壊す側」を担当させることで:

- Claude が見逃しやすいエッジケースを GPT-5.4 が見つける（逆もある）
- 「書く人」と「壊す人」の分離は、人間のチーム開発でも基本原則
- モデル間の得意分野の違いが、レビューの網羅性を高める

## FAQ

### Q: OpenAI APIの料金は別途かかる？
A: はい。Codex CLIの利用はOpenAIの課金です。Claude CodeとCodexで別々のAPI課金になります。

### Q: Codex側がファイルを書き換えることはある？
A: `/codex:rescue` はデフォルトで書き込み可能（`--write`）です。`/codex:review` と `/codex:adversarial-review` はレビューのみで、ファイルを変更しません。

### Q: Claude Codeのプラグインはどこから入れる？
A: Claude Codeのプラグインマーケットプレイスからインストールできます。`openai-codex` プラグインを探してください。

### Q: レビュー結果をClaudeが勝手に修正し始めない？
A: 修正しません。レビュー結果提示後は必ずユーザーに確認を取る設計になっています。どの指摘を修正するかはあなたが決めます。
